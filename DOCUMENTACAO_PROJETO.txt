================================================================================
                    CINEGENIO - DOCUMENTAÇÃO DO PROJETO
================================================================================

VISÃO GERAL
-----------
CineGênio é uma plataforma web de entretenimento pessoal que combina recursos de 
streaming, recomendações personalizadas e acompanhamento de filmes e séries. O 
projeto utiliza tecnologias modernas para oferecer uma experiência rica e 
interativa aos usuários.

================================================================================
TECNOLOGIAS PRINCIPAIS
================================================================================

FRAMEWORK E LINGUAGEM
---------------------
- Next.js 15.5.9 (React 19.1.0)
- TypeScript 5
- Node.js com npm
- Turbopack (modo de desenvolvimento e build)

-----------

- Framer Motion 12.23.12 (animações)
- Lucide React (ícones)

------------------------------
- Firebase 12.1.0 (client-side)
- Firebase Admin 13.4.0 (server-side)
- Firebase Authentication (email/senha)
- Cloud Firestore (banco de dados NoSQL)

APIs EXTERNAS
-------------
- TMDB API (The Movie Database) - Metadados de filmes e séries

- Google Generative AI / OpenAI - Chat AI e recomendações

BIBLIOTECAS ADICIONAIS

================================================================================
SISTEMA DE CACHE E ATUALIZAÇÃO AUTOMÁTICA
================================================================================

ESTRATÉGIA DE CACHE
-------------------
O projeto utiliza um sistema de cache em três camadas:

1. Memória (in-memory cache)
   - Cache temporário durante a sessão
   - Mais rápido, mas volátil

2. Firebase Firestore (collection 'public')
   - Cache persistente compartilhado
   - Documentos com expiração (expiresAt)
   - Tipos de cache:
     * top10-netflix, top10-prime, top10-disney, top10-hbo, top10-apple
     * global-movies, global-series
     * trending, now-playing, popular-movies, popular-tv, on-the-air, upcoming

3. API Externa (TMDB, FlixPatrol)
   - Fonte de dados primária
   - Acessada apenas quando cache expira

VALIDADE DO CACHE
------------------
- Top 10 Streamings: 30 minutos
- Carrosséis TMDB: 1 hora
- Calendário: 3 horas (futuro)

CRON JOB DE ATUALIZAÇÃO
------------------------
Endpoint: /api/cron/update

Funcionalidades:
1. Verifica staleness (antiguidade) de todos os caches
2. Prioriza atualizações:
   - Prioridade 1: Top 10 streamings (mais crítico)
   - Prioridade 2: Carrosséis TMDB
3. Atualiza apenas caches expirados
4. Enriquece dados com TMDB (posters, sinopses, avaliações)
5. Filtra temporadas iniciadas para séries

Processo de atualização:
1. Busca dados da FlixPatrol API (format=id para leveza)
2. Para cada item, busca detalhes completos no TMDB
3. Filtra temporadas de séries (remove temporadas futuras)
4. Salva no Firebase com timestamp e expiração
5. Retorna log de atualizações

Arquivos principais:
- src/app/api/cron/update/route.ts
- src/lib/services/cronUpdateService.ts
- src/lib/services/flixpatrolService.ts

POPULAÇÃO INICIAL
-----------------
Endpoint: /api/populate-radar

Executado automaticamente quando:
- Usuário acessa o dashboard pela primeira vez
- Caches não existem no Firebase

Popula:

- Popular Movies/TV
- On The Air (séries no ar)

INTEGRAÇÃO COM APIs EXTERNAS

1. TMDB (THE MOVIE DATABASE)
-----------------------------
URL Base: https://api.themoviedb.org/3

Endpoints utilizados:
- /search/movie - Busca de filmes
- /search/tv - Busca de séries
- /search/multi - Busca geral
- /movie/{id} - Detalhes de filme
- /tv/{id} - Detalhes de série
- /movie/upcoming - Próximos lançamentos
- /movie/now_playing - Nos cinemas
- /movie/popular - Filmes populares
- /tv/on_the_air - Séries no ar
- /tv/popular - Séries populares
- /trending/all/week - Tendências da semana
- /discover/movie - Descobrir filmes
- /discover/tv - Descobrir séries

Recursos obtidos:
- Metadados (título, sinopse, data de lançamento)
- Imagens (posters, backdrops)
- Avaliações e popularidade
- Gêneros
- Elenco e equipe
- Provedores de streaming (watch providers)
- Temporadas e episódios (séries)

Rate Limiting:
- Fila de requisições com delay de 250ms entre chamadas
- Evita bloqueio por excesso de requisições

Arquivo principal:
- src/lib/tmdb.ts

2. FLIXPATROL API
-----------------
URL Base: https://top-10-streamings.onrender.com

Endpoints utilizados:
- /api/quick/overall?format=id - Top 10 de todos os streamings (IDs)
- /api/quick/overall - Top 10 completo com dados
- /api/quick/calendar/movies - Calendário de filmes
- /api/quick/calendar/tv-shows - Calendário de séries
- /api/quick/calendar/overall - Calendário geral
- /api/firebase/history/{service}/{type}/{date} - Dados históricos

Autenticação:
- Header: X-API-Key
- Chave armazenada em variável de ambiente

Dados obtidos:
- Rankings de streaming services (Netflix, Prime, Disney+, HBO, Apple TV+)
- Índice de popularidade
- Top 10 global (filmes e séries)
- Calendário de lançamentos
- Histórico de rankings

Arquivo principal:
- src/lib/services/flixpatrolService.ts

ESTRUTURA DO FIREBASE
================================================================================

COLLECTIONS PRINCIPAIS
-----------------------

1. users/{userId}
   - name: string
   - username: string
   - email: string
   - createdAt: timestamp
   - photoURL?: string

2. users/{userId}/nowWatching/{titleId}
   - id: number (TMDB ID)
   - type: 'movie' | 'tv'
   - title: string
   - posterUrl: string
   - timestamp: number (progresso em segundos)
   - duration: number (duração total)
   - viewed: boolean
   - lastWatched: timestamp
   - episodeId?: string (para séries)
   - seasonNumber?: number
   - episodeNumber?: number

3. users/{userId}/nowWatching/episodes/{episodeId}
   - Mesma estrutura, para episódios de séries

4. users/{userId}/watched/{titleId}
   - id: number
   - type: 'movie' | 'tv'
   - title: string
   - posterUrl: string
   - rating?: number (0-10)
   - comment?: string
   - watchedAt: timestamp
   - isFavorite?: boolean

5. users/{userId}/watchlist/{titleId}
   - id: number
   - type: 'movie' | 'tv'
   - title: string
   - posterUrl: string
   - addedAt: timestamp
   - priority?: 'high' | 'medium' | 'low'

6. users/{userId}/dropped/{titleId}
   - id: number
   - type: 'movie' | 'tv'
   - title: string
   - posterUrl: string
   - reason?: string
   - droppedAt: timestamp

7. users/{userId}/radar/{titleId}
   - Títulos no radar de lançamentos
   - Estrutura RadarItem completa

8. users/{userId}/collections/{collectionId}
   - name: string
   - description: string
   - items: array de titleIds
   - createdAt: timestamp

9. public/{cacheType}
   - items: array de RadarItem
   - lastUpdated: timestamp
   - expiresAt: timestamp
   - cacheType: string

10. public/highlights
    - items: array de HighlightItem
    - lastUpdated: timestamp

================================================================================
FLUXOS PRINCIPAIS
================================================================================

1. FLUXO DE AUTENTICAÇÃO
-------------------------
1. Usuário acessa a aplicação (/)
2. AuthContext verifica estado de autenticação
3. Se não autenticado, redireciona para /login
4. Usuário faz login ou registro
5. Firebase Authentication valida credenciais
6. Perfil criado/atualizado no Firestore
7. Redirecionamento para /dashboard

2. FLUXO DE VISUALIZAÇÃO DE CONTEÚDO
-------------------------------------
1. Usuário navega pelo dashboard
2. Clica em um título (filme ou série)
3. EnhancedDetailsModal é aberto com detalhes
4. Usuário pode:
   a) Adicionar à watchlist
   b) Marcar status (Assistindo, Assistido, etc.)
   c) Dar nota e comentar
   d) Iniciar reprodução
5. Se iniciar reprodução:
   a) VideoPlayerModal é aberto
   b) Progresso é salvo automaticamente
   c) Ao atingir 90%, marca como assistido
   d) Para séries, sistema de episódio duplo

3. FLUXO DE ATUALIZAÇÃO DE CACHE
---------------------------------
1. Cron job executa /api/cron/update periodicamente
2. Sistema verifica staleness de todos os caches
3. Identifica caches expirados
4. Para cada cache expirado:
   a) Busca dados da API externa
   b) Enriquece com TMDB
   c) Processa e limpa dados
   d) Salva no Firebase
5. Retorna log de atualizações

4. FLUXO DE RECOMENDAÇÃO AI
----------------------------
1. Usuário acessa /dashboard/chat ou /ai
2. Sistema carrega histórico de visualizações
3. Usuário faz pergunta ou pede recomendação
4. Sistema envia contexto para IA:
   - Títulos assistidos
   - Gêneros favoritos
   - Avaliações
5. IA processa e retorna recomendação
6. Sistema busca detalhes no TMDB
7. Exibe recomendações formatadas

5. FLUXO DE CONTINUAR ASSISTINDO
---------------------------------
1. Dashboard carrega nowWatching do usuário
2. Para cada título em andamento:
   a) Verifica progresso
   b) Calcula porcentagem assistida
   c) Para séries, identifica próximo episódio
3. Exibe em ContinueWatchingCarousel
4. Ao clicar:
   a) Abre player no timestamp salvo
   b) Continua reprodução
